# Skills Smart Manager

**Meta-skill for intelligent skill lifecycle management in Claude Code**

## Overview

The Skills Smart Manager is a meta-skill that monitors, analyzes, and optimizes all other skills loaded in your Claude Code sessions. It acts as an orchestrator for your entire skill ecosystem, preventing context bloat, detecting conflicts, and ensuring only relevant skills remain active.

## What It Does

### Core Capabilities

1. **Context Bloat Prevention** — Automatically detects and unloads skills that haven't been used in >15 interaction turns
2. **Dependency Management** — Ensures prerequisite skills are active when needed
3. **Smart Recommendations** — Suggests skills based on project files detected (e.g., `Cargo.toml` → Rust skills)
4. **Health Checks** — Verifies MCP servers and API endpoints are responding
5. **Garbage Collection** — Cleans up temporary files generated by other skills
6. **Conflict Detection** — Identifies contradictory skills (e.g., multiple formatters with conflicting rules)
7. **Session Memory** — Archives optimal skill configuration per repository for faster future loading

## When to Use

### Automatic Triggers

- Context window >60% and ≥3 skills loaded
- User switches project domains ("done with design, let's code")
- Session feels slow or unresponsive
- Rate limit warnings appear

### Manual Invocation

```bash
/skills-smart-manager
```

Or mention these phrases:

- "optimize my skills"
- "clean up loaded skills"
- "why is this session slow?"
- "skill health check"
- "free up tokens"

## Installation

```bash
# Copy to your Claude skills directory
cp -r skills/skills-smart-manager ~/.claude/skills/

# Or symlink (recommended for development)
ln -s "$(pwd)/skills/skills-smart-manager" ~/.claude/skills/
```

## Quick Start

### Basic Usage

```bash
# Scan current session and get recommendations
python3 ~/.claude/skills/skills-smart-manager/scripts/analyze_context.py --action scan

# Unload a specific skill
python3 ~/.claude/skills/skills-smart-manager/scripts/analyze_context.py --action unload --skill diagnose

# Check health of skill dependencies
python3 ~/.claude/skills/skills-smart-manager/scripts/analyze_context.py --action health-check

# Clean up temporary files
python3 ~/.claude/skills/skills-smart-manager/scripts/analyze_context.py --action gc

# Archive session memory for this repo
python3 ~/.claude/skills/skills-smart-manager/scripts/analyze_context.py --action archive
```

## How It Works

### Analysis Engine

The core Python script (`scripts/analyze_context.py`) performs:

1. **Project Type Detection** — Analyzes files in working directory
2. **Skill Usage Tracking** — Monitors which skills are invoked
3. **Token Footprint Estimation** — Calculates approximate token cost per skill
4. **Conflict Detection** — Checks active skills against conflict rules
5. **Recommendation Generation** — Produces actionable optimization suggestions

### Conflict Resolution

Rules defined in `references/conflict-rules.md`:

- **Formatters** — Keep highest priority (Prettier > Standard)
- **Test Frameworks** — Keep whichever is in `package.json`
- **Language Skills** — Keep matching detected project type
- **Deployment** — Keep matching deployment config

### Session Memory

Creates `.skill-memory.json` in project root:

```json
{
  "repository": "/Users/user/project",
  "project_type": "rust-cargo",
  "recommended_skills": ["tdd-hybrid", "rust-best-practices"],
  "avoid_skills": ["diagnose", "vercel:deploy"],
  "last_updated": "2026-05-28T18:45:00Z"
}
```

Next session in this repo auto-loads optimal skills.

## Configuration

### Adjust Staleness Threshold

Default: 15 turns without invocation

```bash
# More aggressive (unload after 10 turns)
python3 scripts/analyze_context.py --action scan --staleness-threshold 10

# More conservative (unload after 30 turns)
python3 scripts/analyze_context.py --action scan --staleness-threshold 30
```

### Enable Auto-Unload (Advanced)

**⚠️ Use with caution** — automatically unloads stale skills without prompting.

Add to `.claude/settings.json`:

```json
{
  "hooks": {
    "on-context-threshold": {
      "threshold": 60,
      "command": "python3 ~/.claude/skills/skills-smart-manager/scripts/analyze_context.py --action scan --auto-unload"
    }
  }
}
```

## Troubleshooting

### Issue: Script returns "Permission denied"

**Fix:**

```bash
chmod +x ~/.claude/skills/skills-smart-manager/scripts/analyze_context.py
```

### Issue: Unloaded a skill I still needed

**Fix:**

Reactivate instantly:

```
Load the [skill-name] skill
```

Or adjust staleness threshold:

```bash
python3 scripts/analyze_context.py --action scan --staleness-threshold 30
```

### Issue: False positive conflict detection

**Fix:**

Edit `references/conflict-rules.md` to remove or adjust the rule.

## Philosophy

**Skills are tools, not decorations.**

Just like you wouldn't keep every tool in your toolbox open on your workbench, you shouldn't keep every skill loaded in your context.

**Load when needed. Unload when done.**

This skill enforces that discipline automatically.
